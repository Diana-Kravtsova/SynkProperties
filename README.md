# Синхронизатор свойств КОМПАС-3D

> **Утилита для аудита и синхронизации свойств деталей в сборочных файлах КОМПАС-3D (.a3d)**

---

## О проекте

При работе со сборками в КОМПАС-3D часто возникает ситуация, когда свойства компонента в сборочнике (наименование, обозначение, примечание) расходятся с реальными свойствами файла детали — из-за ручной правки, переименований или рассинхрона спецификации. Поиск таких несоответствий вручную по большим сборкам занимает много времени и требует внимательности.

Эта утилита автоматизирует весь процесс: открывает сборку, обходит все входящие компоненты, сравнивает свойства и наглядно показывает расхождения. Для штоков и гильз дополнительно проверяется корректность длины в примечании относительно обозначения. Найденные несоответствия можно исправить прямо из интерфейса — одним нажатием данные из файла детали перетянутся в сборочник.

---

## Функциональность

### Сканирование сборки
- Открывает файл сборки КОМПАС-3D (`.a3d`) через COM API
- Обходит все компоненты, **пропуская библиотечные детали** (`C:\Program Files\ASCON`, `C:\ProgramData\ASCON`)
- Считывает свойства компонента из **двух источников**: из записи в спецификации и из файла детали (если файл доступен)
- Выполняет сканирование в **отдельном потоке**, не блокируя UI; сканирование можно прервать кнопкой «Стоп»

### Проверки и диагностика
| Проверка | Описание |
|---|---|
| **Обозначение** | Значение в сборочнике vs значение в файле детали |
| **Наименование** | Значение в сборочнике vs значение в файле детали |
| **Примечание** | Значение в сборочнике vs значение в файле детали |
| **Имя файла** | Соответствие имени файла шаблону `{Обозначение} {Наименование}` |
| **Длина (шток/гильза)** | Сравнение числа длины из обозначения и из поля примечания |

Длина проверяется **только для типовых деталей** (конфигурируется):
- **Шток** — длина на 3-й позиции 
- **Гильза** — длина на 4-й позиции

### Управление конфликтами
- Каждый конфликт отображается таблицей с цветовой индикацией:
  - 🟢 — совпадает
  - 🔴 — ошибка (расхождение свойств или длины)
  - 🟡 — предупреждение (несоответствие имени файла)
- Для каждого конфликта — радиокнопки: «Обновить из детали» / «Не менять»
- Кнопки «Выбрать всё» / «Снять все» для массового выбора

### Применение изменений
- Записывает выбранные данные из файла детали в свойства компонента в сборке
- Обновляет строку спецификации (поля 4, 5, 6: обозначение, наименование, примечание)
- Сохраняет сборку после применения изменений
- Запускает повторное сканирование для верификации результата

### Экспорт в Excel
Выгружает полную информацию по **всем деталям** сборки (не только конфликтным) в `.xlsx`:
- Тип, имя файла, свойства детали и сборочника
- Данные о длине для штоков и гильз
- Ячейки с несовпадениями подсвечены: 🔴 красным (ошибки), 🟡 жёлтым (имя файла), 🟢 зелёным (совпадает)
- Автоподбор ширины столбцов, фиксация заголовка

## Требования

- **КОМПАС-3D v24** (или совместимая версия с COM API `KOMPAS.Application.7`)
- **Windows** (COM API доступен только на Windows)
- Python 3.8+

### Зависимости

pip install customtkinter pywin32 openpyxl

## Конфигурация типов деталей

Логика распознавания типов деталей и позиций длины задаётся в словаре `PART_TYPE_CONFIG` в начале файла:

```python
PART_TYPE_CONFIG = {
    "RD": {"length_position": 2, "description": "Шток"},  
    "TB": {"length_position": 3, "description": "Гильза"},
}
```

Чтобы добавить новый тип — достаточно добавить новую запись с нужным префиксом и индексом позиции числа длины в обозначении.

Пути к стандартным библиотекам КОМПАС, компоненты из которых пропускаются при сканировании, задаются в `LIBRARY_PATHS`.
